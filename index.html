AI / Agent Systems

LLM Control Planes
Central routing, policy, and cost-optimization layer that sits in front of multiple LLM providers. Handles model selection, fallbacks, rate limits, auth, prompt/response logging, and per-tenant quotas — turning a collection of model APIs into a governed platform surface.
LangChain / LangGraph
Frameworks for composing LLM-powered applications. LangChain provides chains, tools, retrievers, and memory primitives. LangGraph layers a graph-based state machine on top — enabling cycles, branching, and multi-agent flows that plain chains cannot express cleanly.
RAG Pipelines
Retrieval-Augmented Generation: ingest documents, chunk, embed, store in a vector DB (+ metadata index), retrieve top-k at query time, and feed into the prompt. Ends with evaluation (RAGAS, DeepEval) on faithfulness, answer relevance, and context precision.
Multi-Agent Orchestration
Coordinating multiple specialized agents (e.g. planner, executor, critic, evaluator) around a shared task. Requires routing, message passing, shared/scoped memory, tool permissions, and deterministic handoff — plus guardrails so one misbehaving agent doesn't blow up the workflow.
Prompt Engineering Frameworks
Structured prompt composition: templates, few-shot example libraries, role/system prompt separation, output-schema enforcement (JSON mode, function calling), and regression testing of prompts like code — so prompt changes don't silently degrade production quality.

Cloud & Infrastructure

AWS — EKS, Lambda, S3, RDS, Bedrock
Core AWS surface: EKS for Kubernetes control plane, Lambda for event-driven compute, S3 for durable object storage, RDS for managed relational DBs, and Bedrock as the managed-LLM entrypoint for foundation models (Claude, Llama, Titan).
Multi-Region HA Architectures
Active-active and active-passive deployments spanning multiple AWS regions. Covers cross-region replication (S3, RDS, DynamoDB global tables), Route53 failover, regional isolation of blast radius, and DR runbooks that are rehearsed, not just documented.
Event-Driven Systems — SNS/SQS, Step Functions
Asynchronous workflows: SNS for pub/sub fan-out, SQS for durable decoupling + retries + DLQs, Step Functions for long-running state machines. Turns synchronous RPC chains into resilient pipelines where each stage can fail and be replayed independently.

DevOps / IaC

Terraform, AWS CDK
Declarative infrastructure-as-code. Terraform for multi-cloud / provider-agnostic work with state management and reusable modules. AWS CDK for AWS-native workflows that benefit from a real programming language (Python/TypeScript) and higher-level constructs.
GitHub Actions, GitOps
CI/CD pipelines defined as YAML next to the code. GitOps takes this further — cluster/infra state is continuously reconciled against what's in Git (ArgoCD, Flux), so the repo is the source of truth and rollbacks are just git reverts.
Kubernetes — EKS, CRDs, Controllers
Container orchestration plus the operator pattern: Custom Resource Definitions extend the K8s API, and controllers watch those resources and drive the world toward declared state. The cleanest way to encode platform logic inside the cluster itself.

Observability

OpenTelemetry
Vendor-neutral instrumentation standard for traces, metrics, and logs. SDKs emit data in a common format; the collector routes it to any backend (Prometheus, Jaeger, Datadog, CloudWatch). Breaks the lock-in between application code and observability vendor.
Prometheus, Grafana
Prometheus pulls and stores time-series metrics with a powerful query language (PromQL). Grafana sits on top for dashboards, alerting, and visual exploration. The de-facto open-source monitoring pair for Kubernetes and cloud-native workloads.
ELK, CloudWatch
ELK (Elasticsearch + Logstash + Kibana) for searchable, structured log aggregation across many services. CloudWatch for AWS-native metrics, logs, and alarms tightly integrated with every AWS service. Often run side-by-side — one for apps, one for infra.

Security & Governance

OPA — Policy-as-Code
Open Policy Agent: a general-purpose policy engine that evaluates Rego rules. Decouples authorization logic from application code so the same policies enforce K8s admission, API authorization, and Terraform plan checks — policy becomes a versioned artifact.
RBAC, Zero Trust
RBAC — access tied to roles and bindings, enforced at every API boundary. Zero Trust — drop the network perimeter assumption; every request is authenticated, authorized, and encrypted regardless of origin. Identity becomes the new perimeter.
AI Guardrails
Input and output controls around LLM calls: prompt-injection defense, PII redaction, content policy filters, hallucination / groundedness checks, and refusal handling. The "hard edges" that make an otherwise-stochastic model safe to put in front of users.
