AEGIS

AI Trust & Evaluation Framework · governance, evaluation, and safety for agentic systems

Live API · Source · Portfolio

About

AEGIS is a governance, evaluation, and safety layer for agentic AI systems. It sits between application code and the LLM/agent runtime, enforcing policy, scoring outputs, and surfacing the evidence trail an auditor or platform team needs.

  • Evaluation pipelines — RAGAS, DeepEval, custom rubrics
  • Policy enforcement — OPA/Rego at request and response boundaries
  • Multi-agent validation — planner / executor / critic with deterministic handoff
  • Observability + scoring — every call traced, scored, and queryable

Architecture

User
  → API Gateway
    → LLM Control Plane
      → Agent Orchestrator
        → Tool Execution
        → Memory (Vector + Metadata)
      → Observability + Evaluation
      → Policy Enforcement (OPA / RBAC)

API

Interactive Swagger UI: aegis.dmiruke.dev/docs

MethodEndpointPurpose
POST/v1/evaluateRun an evaluation rubric against an LLM trace
POST/v1/policy/checkEvaluate a request/response against active OPA policies
GET/v1/scores/{trace_id}Retrieve scores for a recorded trace
GET/v1/policiesList active policies
GET/healthzLiveness probe

Endpoints are illustrative — canonical spec at /openapi.json.

Quick Start

# Hit the live API
curl https://aegis.dmiruke.dev/healthz

# Evaluate a trace
curl -X POST https://aegis.dmiruke.dev/v1/evaluate \
  -H "Content-Type: application/json" \
  -d '{"trace_id": "demo", "rubric": "faithfulness"}'

# Or run it locally
git clone https://github.com/dattamiruke/aegis.dmiruke.dev
cd aegis.dmiruke.dev
docker compose up

Core Stack

  • Python, FastAPI, Docker
  • OPA (Rego) for policy-as-code
  • RAGAS, DeepEval for LLM evaluation
  • OpenTelemetry, Prometheus for observability
  • Hetzner + nginx (TLS via Let's Encrypt)
